In order to further investigate failed requests, you can look in the following places:
  • Event Logs
    Use Event Logs to see the exception that caused the failed request along with stack trace. Using the stack trace, you can identify the part of your application that is causing the failed requests.
  • Failed Request Tracing Logs
    These logs will give you detailed information about any failed requests. They include the trace of all IIS components that are used to process the request and the time taken of each component. be useful if you are attempting to increase site performance or isolate what is causing a specific HTTP error to be returned.
  • Application Insights
    Use Application Insights if you are using an ASP.NET application. If you want to investigate uncaught exceptions that are causing failed requests, Application Insights makes it very easy to capture any unhandled exception
